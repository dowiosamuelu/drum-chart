# 鼓手攻略表（Drum Chart）

把一首歌按**當天曲序**排成段落（Intro / Verse 1 / Verse 2 / 副歌 / Bridge…），
每段標清楚**打什麼節奏（groove）、用什麼過門（fill）**，
然後把整份**分享給其他鼓手**——對方能看、能聽，還能**做自己的版本**。

目標是慢慢長出一個「鼓手互相觀摩的知識庫」。

---

## 怎麼啟動

這是純前端的單檔網頁，但**不能直接雙擊開**（`file://` 會讓 YouTube 出現錯誤 153，
而且自架的 GrooveScribe 也需要用伺服器開）。請用一個本機小伺服器：

```bash
cd drum-chart
python3 -m http.server 8777
```

然後瀏覽器開：**http://localhost:8777/index.html**

> 關掉終端機伺服器就停了，下次要用再跑一次同一行指令即可。

---

## 怎麼用

1. **新增歌曲**：右上角「＋ 新增歌曲」。填歌名、你的名字（編輯者）。
2. **排曲序**：右側「＋ 新增段落」一段一段加；用每段左邊的 ▲▼ 調整順序。
3. **填每段的打法**：點段落的「編輯」，貼上
   - **節奏 groove 連結**
   - **過門 fill 連結**（選填）
   - **備註**（選填，例如「副歌前漸強」「鈸收尾」）
4. **怎麼生出 groove 連結**：點「開自架 GrooveScribe 編輯」→ 在裡面敲好你的節奏 →
   **複製網址列整段連結** → 貼回段落欄位。
5. **看＆聽**：點任一段落，右下會顯示這段的節奏與過門格子譜，可播放、可放大。
6. **參考影片（選配）**：左邊可貼 YouTube 連結聽原曲 feel。
   注意現場曲速常和 YT 不同，影片只是參考、不是時間軸主軸。

---

## 怎麼分享

短期用**檔案**分享（不用網路、不用帳號）：

- **匯出這首**：存成 `歌名.json`。
- **匯出全部**：把所有歌存成 `本週歌單.json`（一週多首一次傳）。
- **匯入檔案**：選 `.json` 匯入，單首或整包都吃。
- **做我自己的版本**：把目前這份複製成「（我的版本）」再改。

把 `.json` 用 LINE / email / 雲端硬碟傳給其他鼓手即可。

---

## 已知限制 / 設計取捨

- **JSON 是快照，不會自動同步**：別人改完傳回來要手動再匯入；各自的版本會各自長。
  等這個「共享→觀摩→做版本」的迴圈確認有價值，下一步才做真正的**雲端資料庫**。
- **資料存在瀏覽器 localStorage**：換瀏覽器／清快取會不見，重要的請記得匯出存檔。
- **鼓聲與 YouTube 無法鎖定同步**：兩個獨立音源可同時響，但會 drift，當「二選一」用。

---

## 技術

- 單檔 `index.html`，純前端，資料存 localStorage。
- 格子譜與播放外包給 **GrooveScribe**（開源，自架在 `groovescribe/`），
  顯示走它的精簡嵌入頁 `GrooveEmbed.html`，編輯走完整 `index.html`。
- YouTube 用官方 IFrame API 嵌入。
