# CLAUDE.md — 鼓手攻略表（Drum Chart）

## 這是什麼
小圈子鼓手共用的「鼓手知識庫」。以**曲序**為脊椎排段落（Intro/Verse/副歌…），每段用內建格子編輯器標**節奏 groove ＋ 過門 fill ＋ 備註**，匯出成 JSON 分享給其他鼓手，對方能看＆聽（真鼓音色），再「做自己的版本」。
真實情境：教會敬拜，主領一週開四首歌；面向華語鼓手，補中文鼓譜資源的空缺。

## 架構（刻意極簡）
- **單檔** `index.html`：純前端、無框架、無後端、無 build。資料存瀏覽器 `localStorage`（key `drumchart_v3`，`load()` 會 `migrate()` 補新軌）。
- **鼓譜格式**：`pattern = { tempo, ri[], oh[], hh[], tm[], sn[], kk[] }`，每軌 16 格（4/4、十六分）。
  - `tm`：0空 / 1 H / 2 M / 3 L（高中低 tom）
  - `sn`：0空 / 1 重音 / 2 ghost
  - 其餘軌：0 / 1
  - 軌順序常數見 `LANE_KEYS` 與 `LANES`。
- **播放**：Web Audio 載入 `samples/*.mp3`（真鼓取樣）觸發；`playBuf()` + `GAINS` 表做各鼓件音量微調（目前 open hi-hat 壓到 0.5）。排程器在 `playToggle()`（25ms lookahead）。
- **分享**：JSON 匯出/匯入（`songPayload` / `importSong`）＋ fork（做我自己的版本）。**不用分享連結**（URL 太長會爆）。
- **YouTube**：選配參考，可收合、lazy mount；**不做**與鼓聲的時間軸同步（現場曲速與 YT 不同，硬對是白工）。

## 重要設計決策（不要回頭推翻）
- 骨架是**曲序**不是時間軸。
- 編輯**就地點格子**，不依賴外部工具（早期版本曾外連 GrooveScribe，已移除；僅借用其取樣音檔放進 `samples/`）。
- 拍號**只做 4/4 / 16 格**；其他拍號是獨立的大工程，MVP 不碰。
- 合成鼓「太假」→ 改用真鼓取樣；要更好就換更高品質 sample，不要走回合成。

## 怎麼跑
`python3 -m http.server 8777` 後開 http://localhost:8777/index.html（不能用 file://）。

## 下一步候選（按價值）
1. **先驗證**：拿去和真實鼓手跑一次 JSON 分享迴圈，確認有價值——這比寫新功能重要。
2. 真正的**雲端資料庫**（取代 JSON 快照、解決不同步）——迴圈確認有價值後才做。
3. 同一首歌**多版攻略並排比較**；每軌音量/拍號擴充；更真的音色。
